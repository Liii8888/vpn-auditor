# vpn-auditor

[English](README.en.md)

> 连上代理之后，它只问网络自己。

`vpn-auditor` 是一个零交互 VPN / 代理 / 梯子体检 skill。你连接好线路，运行它，然后等一份直接的中文结论。它不会让你手动断网、解释客户端、等晚高峰，或者打开一排网页做人工确认。

```text
结论：87/100。好，日常很稳，未命中一票否决。
```

```text
VPN ON
  -> 出口探针
  -> DNS / IPv6 / 路径检查
  -> 公开目标连通性
  -> 分数、证据、影响因素
```

## 一次运行

| 你做 | 它做 | 你看 |
| --- | --- | --- |
| 连好 VPN / 代理 | 自动探测出口、DNS、IPv6、路径和连通性 | 总分、结论、一票否决 |
| 不切网络 | 不做破坏性测试 | 自动检测证据 |
| 不解释客户端 | 不做人肉审查 | 影响因素和未覆盖项 |

## 报告长什么样

| 区块 | 读法 |
| --- | --- |
| 结论 | 先看这一行。能不能当主力，直接说。 |
| 评估摘要 | 用“强 / 稳 / 有短板 / 偏弱”扫一眼各类表现。 |
| 自动检测证据 | 看出口 IP、DNS、IPv6、隧道接口、目标可达性。 |
| 影响因素 | 看为什么被扣印象分，但不公开详细权重表。 |
| 本轮未覆盖 | 看哪些项目因为需要人工或破坏性测试而跳过。 |

## 它会追问网络

| 身份 | 路径 | 体验 |
| --- | --- | --- |
| 公网出口 IP | 默认 DNS / scoped DNS | 公开目标可达性 |
| 多探针一致性 | IPv6 证据 | 小样本响应 |
| 本地隧道 / 系统代理 | 本地网络 DNS 迹象 | 连续请求稳定性 |

## 它不会折腾你

| 不做 | 为什么 |
| --- | --- |
| kill switch / 强制断网 | 会打断你当前网络 |
| 客户端来源审查 | 需要人工判断或额外权限 |
| 证书、描述文件、内核扩展、MDM 检查 | 不适合零交互体检 |
| 服务商商业模式判断 | 一次网络探测证明不了 |
| 长期高峰期稳定性 | 需要多时段监控 |
| 银行、校园网、登录类站点 | 可能触发风控或碰到隐私 |

## 安装

```bash
mkdir -p "$HOME/.codex/skills"
cp -R vpn-auditor "$HOME/.codex/skills/vpn-auditor"
```

如果 Codex 的 skill 列表没有刷新，重启 Codex。

## 使用

连接好 VPN / 代理后，调用 `$vpn-auditor`。也可以直接运行脚本：

```bash
python3 "$HOME/.codex/skills/vpn-auditor/scripts/vpn_auditor.py"
```

离线自测：

```bash
python3 "$HOME/.codex/skills/vpn-auditor/scripts/vpn_auditor.py" --self-test
```

## 设计原则

- 只测能自动证明的东西。
- 不把用户变成测试员。
- 不打印详细权重表，只给结论、证据和影响因素。
- 算法仍然可审计，实现在 `scripts/vpn_auditor.py`。
